Embed Partner Onboarding v 2.0¶

Tip

Your feedback is welcome and is a key driver as documentation evolves. Send suggestions to acrobatsignembed@adobe.com.

Both the partner and the Acrobat Sign team can onboard to Acrobat Sign Embed 2.0 by implementing a one-time setup. Partners first onboard themselves in sandbox environment so they can learn how to create accounts in Sign, add users to these accounts, and use Sign via the REST v6 APIs. Once they are comfortable with the new Embed 2.0 APIs, partners can proceed with onboarding to the production environment. The steps for onboarding to the sandox and production enviroment are similar.

  • Sandbox: https://secure.adobesignsandbox.com/

  • Production: https://secure.adobesign.com/

Note

Before using Sign Embed services in production, your application must undergo formal certification by Adobe. Once your development is complete, you will collaborate with your Product Success Manager to get your application certified.

Parent organization set up¶

Adobe initiates the organization setup process as follows:

  1. You provide the name and email address of up to two administrators to your Product Success Manager.

  2. Adobe creates your organization within the Admin Console, adds your admin contact information, and assigns global and system admin roles.

Note

When the process completes, your admin(s) receive 2 email invitation emails: one each for the Admin and Global Consoles.

Create an admin account¶

After Adobe creates your organization, create an admin account:

  1. Open the invitation Admin Console email and accept the invitation.

  2. Create an account with your admin email. You’ll need to verify your identity.

Configure your organization¶

The admin must claim a domain and link it with a federated directory. It allows you to create users with the claimed domain and add them in Embed accounts.

  1. Log in to the https://adminconsole.adobe.com with the credential you created above.

  2. Go to Settings > Identity Settings > Domains.

  3. Choose Add Domain.

  1. Specify a domain.

  2. Choose Next.

  1. Choose Add Domain.

  2. Choose Next.

  3. Choose Validate.

  1. Copy the key string. For example: @ 3600 IN TXT adobe-idp-site-verification=dde4f6d5eaf2089613c7acc16610593a74560d54a51c22b4d33bbb72

  1. Claim your domain.

Note

It may take some time for the DNS change to propagate so that Adobe can validate the domain ownership (an hour is possible). If you add the DNS settings but get an error, try again later as the DNS settings might not yet be updated on the web.

  1. After the domain claiming process completes, choose Validate Now.

Create and link directories¶

  1. If you are in a new session, re-login to the Admin Console.

  2. Go to Settings > Identity Settings > Directories.

  3. Choose Create Directory.

  4. Enter a directory name.

  5. Select an ID type. There are two types of directory types available:

  • Federated ID: Supports Single Sign-On (SSO) via an Identity Provider. The organization hosts and maintains the user accounts and their assets.

  • Enterprise ID: Allows the organization to maintain user accounts and assets, while Adobe takes care of the authentication. This option does not support SSO. For Adobe to authenticate and maintain user credentials, the users receive a password setup email through which they can setup their login password.

  1. Select Next.

Tip

You may need to setup an Identity Provider (IDP). It is not required if only customer users are created with the claimed domain. It is required where the partner admin adds users with the claimed domain in the parent organization and provides them with entitlements to other Adobe products.

  1. To set up an IDP:

  • If you select one of the default IDPs, choose Next and complete the workflow.

  • If you select one of the Other SAML providers, choose Next and then choose Cancel from the following screen. When you see the “Auto-account creation will be enabled” dialog, choose Confirm.

  1. Review your domain, and choose Link Directory.

  1. Select a directory to link to your domain. If there aren’t any, choose New Directory.

  2. Choose Link.

  1. In your domains list, select a domain.

  2. Choose Add domain.

Create a child organization¶

The following steps occur in the Global Admin Console.

The Global Admin Console facilitates managing users since you can create child organizations and segregate users, domains, products, and so on. The child organization resides beneath the parent organization hierarchy. The child org is where you will create your technical account and your customer. This hierarchy keeps your customer accounts and users separate from your company’s users.

  1. Go to https://global-admin-console.adobe.com/. You can also click the Global Admin Console icon.

  1. Log in if you you’re not already.

  1. Choose Policies.

  2. If the parent organization is not selected by default, select it.

  1. Enable the Inherit system admins from parent when child organization is created policy under Policies tab in the Parent Org.

    This setting will add the parent org admin to the child organization(s) at the time of the organization’s creation as a system admin.

  2. Click the plus icon.

  1. Enter a child organization name.

  2. Specify a country or region.

  3. Choose Save.

  1. Choose Review pending changes in the upper right.

  2. Choose Submit changes in the upper right.

  1. Review your settings.

Allocate transactions¶

Product allocation to the child organization is required so that it has access to the Acrobat Sign Embed offer and the technical account is able to provision to Sign. The minimum transactions required for allocating a product is 1.

Tip

The technical account for the child organization will be allowed to exceed the transactions allocated.

  1. Choose Product Allocation in the top bar.

  2. Choose the plus icon under Grant for the child organization you just created.

  3. Allocate at least 1 transaction of the Acrobat Sign Embed product.

  1. Choose Save.

  2. Choose Review pending changes in the upper right.

  3. Choose Submit changes in the upper right.

  1. Verify your configuration

    1. Go to the Admin Console.

    2. Select the organization drop-down list in the top bar.

    3. Choose the child organization you just configured.

  1. Verify your transaction allocation.

Create a technical/developer account¶

Note

These steps require system admin privileges. The child organization may have users with developer role, but only admin users have access to add the requisite API SDKs.

You will have received an email to access the admin console for the child org.

  1. Open the email and select Get started.

  2. During login, join the team and select the child organization.

  3. Navigate to the developer console and login using your admin credentials.

  4. Verify that you are logged into the child organization. If not, select it from the navigation bar.

  5. From the upper-rigth corner, select the child organization you’re configuring from.

  1. Select Create New Project.

  2. Select Add an API

Tip

You add two SDKs and repeat the process for the User Management API.

  1. Select the Adobe Acrobat Sign Embed API SDK. It provides the partner with privileges to call the Sign Embed service APIs, create Acrobat Sign accounts, and add users.

  2. Select Next.

  3. Select OAuth Server-to-Server.

  1. Select Save Configured API.

  2. Select Generate Access Token.

A screen appears with the requisite developer details.

  1. Select Add to Project > API.

  2. Repeat the steps above for the User Management API. This SDK provides admin console access for user creation via the API.

  3. Select Next .

Register the partner via the API¶

You can now use your technical account token to register with Acrobat Sign via the register partner API. You must use the Register API to register your application in both production and sandbox.

Provide account info to Adobe¶

Once the above steps are complete, provide the following information to your Product Success Manager:

  • Client ID

  • Technical Account ID

  • Technical Account Email

  • Organization ID

Your Product Success Manager will set up a Channel and include your technical account in it. If you have sandbox access, they will also help activate your application in the sandbox environment, allowing you to utilize the Sign Embed services there. However, it’s important to remember that full access to the Sign Embed services in the production environment requires formal certification of your application by Adobe.

Onboarding your customers¶

You can now use the new OEM 2.0 APIs to create accounts and add users.
